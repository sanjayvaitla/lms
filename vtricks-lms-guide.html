🔑 Forgot / Reset Password Flow

1. User clicks "Forgot password?"

On login page → redirected to /forgot-password

2. POST /auth/forgot-password { email }

Server generates 32-byte random token → SHA-256 hash stored in DB → 30 min expiry → branded email sent with reset link

3. User clicks email link

Opens /reset-password?token=xxxx → form with password strength meter + confirm field

4. POST /auth/reset-password { token, password }

Server: verify hash → check not used + not expired → bcrypt new password → mark token used → revoke all refresh tokens → force re-login

5. ✅ Success

Redirect to /login with success state

🌐

Google OAuth 2.0

Handled by Passport.js. Redirects to Google → callback → issues JWT pair → redirect to frontend.

NEEDED TO ACTIVATE:

GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=xxx
GOOGLE_CALLBACK_URL=https://yourdomain.com/api/v1/auth/google/callback
📱

MSG91 OTP (MFA)

Phone-based auth. POST /send-otp → 6-digit OTP via SMS → POST /verify-otp → JWT issued.

NEEDED TO ACTIVATE:

MSG91_API_KEY=your_api_key
MSG91_TEMPLATE_ID=your_template_id
MSG91_SENDER_ID=VTRICK
# Requires DLT registration for India

☁️ File Storage & S3

server/src/lib/storage.ts — auto-switches between S3 and local disk.

✅ S3 Already Wired — Zero Extra Code Needed

Set the 4 env vars below → restart server → all uploads go to S3 automatically. /health endpoint confirms which mode is active.

🏠 Local Mode (default)

Files saved to server/uploads/. Served via /uploads/... static route.

AWS env vars not set → auto local mode. Good for development.

☁️ S3 Mode

Files uploaded to S3. URLs returned as public or presigned URLs. Delete also hits S3.

All 4 AWS env vars set → auto S3 mode. Production ready.

🔧 Activate S3 — 5 Minutes

AWS_S3_BUCKET=vtricks-lms-prod
AWS_REGION=ap-south-1# or us-east-1, etc.
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
AWS_S3_CUSTOM_DOMAIN=https://cdn.vtricks.com# optional CloudFront CDN

🚀 Deployment Guide

Steps to push Vtricks LMS to production.

🖥️

Backend Options

Railway, Render, AWS EC2/ECS, DigitalOcean App Platform, Fly.io. Needs: Node.js 18+, PostgreSQL connection string.

🌐

Frontend Options

Vercel, Netlify, AWS S3 + CloudFront, Render. Run pnpm build in client/ → deploy dist/ folder.

🗄️

Database Options

Supabase, Railway PostgreSQL, AWS RDS, Neon.tech. Just update DATABASE_URL. Run pnpm db:schema once.

📧

Email Options

SendGrid (free tier: 100/day), AWS SES (cheap at scale), Gmail SMTP (dev only). Set via env vars — no code change.

📋 Pre-Deploy Checklist

1. Set all production env vars

DATABASE_URL, JWT secrets, CLIENT_URL, CORS_ORIGIN, AWS S3 vars, email config

2. Run DB schema

cd server && pnpm db:schema — creates all 19 tables and applies migrations

3. Run DB seed (optional)

pnpm db:seed — creates demo admin user + sample data

4. Build server

cd server && pnpm build → compiles TypeScript to dist/

5. Build client

cd client && pnpm build → outputs to client/dist/

6. Set NODE_ENV=production

Enables SSL for DB, disables SQL logging, uses 'combined' Morgan format

7. Verify /health endpoint

Should show: {"status":"ok","storage":"s3","google":"configured"}
